Unsure of how to proceed?

Running Otis:

  • Import the dataset.
  • Import command accepts a string specifying a local file or url, or it will ask for this information if nothing is provided.
  • View or clear the data, or import new data.
  • View accepts an integer specifying how many rows of data to display, or will display all rows if nothing is provided.
  • Train the network.
  • Train accepts an integer specifying the output row, or will ask for the output row if nothing is provided.
  • Run the network.

Not familiar with Python and/or Git?

For the list of commands available on the shell, see below.

Using the Shell | Commands

After cloning the Otis repository to your local machine, the simple command python Shell.py will get the shell up and running.

Once the network is up and running, choose from one of these commands to try out the network:

  • import [source] : Imports data from source (URL or File)
  • export [destination] : Exports data to a CSV
  • clear : Clears current data
  • view [rows] : View [rows] rows
  • train : Trains network based on imported data
  • run [data] : Get output from network
  • help : Show this message
  • exit : Exit Otis

Background of Neural Networks

Like the neurons in our brain, artificial neural networks (ANNs) attempt to mirror the complexity and follow similar learning patterns to humans.

Image source

History

ANNs have been around since the early 1900s , centering around perceptrons. A perceptron is a neuron (artificial of course) that works with binary I/O, taking in 1's and 0's and outputting either a 1 or 0. While this neuron seems limited with its restriction to binary, the different aspects of ANNs such as weights and biases with layers allow for intricate learning patterns. Below are these aspects described in more detail:

Terminology


Inputs/Outputs: Every network begins with a certain set of inputs, inputs that change when moving across the network to eventually become outputs; outputs that will most likely be run through the network over and over, hopefully improving in accuracy each time.
Layers: A network has multiple layers with each layer containing a column of nodes and each node holding weights and biases.
Weights: As seen in the diagram above, the inputs will travel from left to right through the different layers. It is the weights that determine what path (line) each input will take. Placing a higher weight to a path will increase the probability of an input taking that specified route.
Bias:When the inputs travel through the network, each node carries a bias, a value that changes as the network learns over multiple iterations, or epochs.
Epochs: An epoch is one iteration through the network. Though the more the merrier usually holds true for epochs, too many can lead to a term called overfitting, when the network iterates so much it ceases to learn and merely follows the pattern laid out before it.
Cost Function: The cost function is implemented to calculate how well the network is learning in relation to what the weights/biases are set at and become.
